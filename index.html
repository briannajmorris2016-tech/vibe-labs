📖 Overview

Welcome to the Vibe Coding Lab. In this hands-on session you'll use vibe coding — building applications by describing what you need in plain language — to create a fully functional Power App for United Nations operations.

What is Vibe Coding?

Rather than writing code, you prompt Power Apps in everyday language. AI generates the data model, screens, and logic. Vibe apps combine agents for UI/logic/data, Dataverse for storage, and Managed Platform for governance.

Lab Scenario

You're a program officer at a UN agency. Inquiries arrive through scattered emails, phone calls, and walk-ins. Requests fall through the cracks. Your mission: build a Program Inquiry & Referral Hub that centralizes intake, categorizes inquiries, assigns them, and tracks resolution.

Learning Objectives

  • Create a Power App from scratch using natural language
  • Iterate and refine through conversational prompts
  • Apply vibe coding to humanitarian use cases
  • Customize themes, layouts, and workflows without code

1️⃣ Exercise 1: Create Your App with a Natural Language Prompt

Step 1–3: Navigate & Select Environment

Open https://vibe.powerapps.com. Confirm the environment name in the bottom-left corner matches your workshop environment.

Step 4–5: Enter Your App Prompt

PROMPT 
Create a centralized inquiry management solution for a United Nations agency that enables staff to capture, triage, and manage inquiries received from partners, NGOs, government ministries, and the public. The solution should allow users to log and categorize inquiries, assign them to responsible staff members, and track their status through resolution. It should maintain structured records for inquiries, partner organizations, staff contacts, and inquiry categories, while providing staff with clear visibility into workload, inquiry priorities, and resolution progress. A dashboard should summarize inquiry status, priority, and volume to support operational oversight and timely response.
TipKeep prompts high-level. Vibe is designed for iterative refinement.

Watch the agents on the left-side chat panel gather requirements, design the data model, generate a plan, and build the app. Expected tables:

PROMPT 
Add an additional 15 rows of sample data to the inquiry table.

2️⃣ Exercise 2: Refine & Extend Your App

Step 10: Light & Dark Mode

PROMPT 
Add a light/dark mode toggle to the app. Add a toggle button (e.g. a sun/moon icon) in the top navigation bar that lets the user switch between light and dark mode.

Step 11: Referral Tracking Fields

PROMPT 
Add a 'Referring Organization' dropdown to the Inquiry form that pulls from the Partner Organizations table. Also add a 'Response Deadline' date field and a 'Follow-up Required' yes/no toggle.

Step 12: Dashboard Screen

PROMPT 
Add a new dashboard screen that shows: the total number of open inquiries, the number of inquiries by category displayed as a bar chart, a list of overdue items where the response deadline has passed, and the number of high-priority inquiries currently unassigned.

Step 13: Expanded Status Workflow

PROMPT 
Update the Inquiry status field to include these options: New, Under Review, Assigned, In Progress, Pending Partner Response, Resolved, and Closed. Add a 'Status History' section on the detail screen that logs when the status changes and who changed it.

Step 14: UN Branding

PROMPT 
Update the app's color theme to use UN Blue (#009EDB) as the primary color and white text for headers. Add the title 'UN Inquiry Hub' as a header banner on all screens. Make the navigation bar consistent and visible on every screen.

Step 15: Search & Filtering

PROMPT 
Add a search bar to the inquiry list screen that searches across subject, description, and partner organization name. Add filter buttons for Status (Open, In Progress, Resolved) and Priority (High, Medium, Low).

Step 16: Urgent Items Highlight

PROMPT 
On the dashboard screen, add an 'Urgent Attention' section that highlights inquiries with High priority that have been in 'New' status for more than 24 hours. Display these in red-bordered cards with the subject, partner organization, and time since received.

Step 17: Team Referral Workflow

PROMPT 
Add a 'Refer to Team' button on the inquiry detail screen. When clicked, it should display a dropdown of available teams: Food Distribution, Livelihoods & Resilience, Nutrition, Logistics & Supply Chain, External Relations, and Emergency Response. When a team is selected, update the inquiry's assigned team and change the status to 'Assigned.'

⭐ Best Practices for Vibe Coding

Crafting Effective Prompts

  • Be specific: "Add a Priority dropdown with options High, Medium, Low" works better than "add a priority field"
  • Name your tables and fields: tell the AI exactly what you want things called
  • Describe relationships: "Link the Inquiry to the Partner Organizations table through a lookup field"
  • One feature per prompt when refining

Iterative Development

  • Start simple, layer on complexity
  • Test after every change
  • Rephrase with more detail rather than starting over
  • Save frequently
