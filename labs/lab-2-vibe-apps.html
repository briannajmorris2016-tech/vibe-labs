Lab Overview

This lab demonstrates how Vibe Power Apps enables you to build a Device Ordering app using natural language guidance, AI-assisted UI generation, and automated logic creation, backed by Dataverse and supported with security and governance.

Core Concepts

Vibe apps bring together:
  • Agents that generate UI, logic, and data (resulting in a full-stack React app)
  • Dataverse as the primary data store
  • Managed platform capabilities for scaled deployment and governance

Concepts (why they matter)

  • Agent-generated UI: accelerates app delivery by generating pages, forms, and logic from natural language.
  • Business logic via natural language: creates validation, conditional formatting, and patterns by interpreting intent.
  • ALM: every Vibe app lives in a Dataverse Solution by default and can be promoted to downstream environments.

Prerequisites

  • Power Apps Premium licensed users
  • Environment where you can create solutions and use Dataverse
  • Tenant prerequisites for Copilot/vibe enabled (see Learn)
Reminder: Vibe is not supported in the Default environment. Ensure participants are using a non-default environment and that the environment name matches in the Vibe UI.

Lab Steps

1
Create a new solution

In Power Apps, create a new solution named Device Ordering Solution.

2
Create a publisher

Create a new publisher named Contoso and click Save.

3
Set as preferred solution

Set Device Ordering Solution as your preferred solution and create as shown in the lab screenshot.

Note: When you set a preferred solution, all components generated by Vibe—including plans, code apps, and data assets— are created inside that solution. Any new components created later will also be added automatically in the same solution.
4
Open vibe.powerapps.com and confirm environment
  1. Navigate to https://vibe.powerapps.com
  2. Confirm the environment name in the bottom-left corner matches the environment where the solution was created.
Tip: Vibe is NOT supported for the Default environment.
5
Review the Vibe home experience

Explore suggested prompts and navigation options:

  • Home: where you start vibing
  • Plans: your past plans created through vibe
  • Apps: list of vibe-created apps
  • Profile & Environment picker: logged in profile + current environment
  • Prompt box: enter your prompt to start creating
  • Submit button: kicks off creation
  • Sample prompts gallery: try sample prompts
6
Submit the prompt to generate the device ordering app
PROMPT 
Create a centralized device ordering solution that allows employees to browse available devices and submit device procurement requests. The solution should provide real-time visibility into the device inventory and status of submitted orders, improving efficiency and transparency across the organization.
Tip: Keep prompts high-level. Vibe is designed for iterative refinement.
7
Observe the agents while they build

Watch the left-side chat panel as agents gather requirements, design the data model, generate a plan, and build the app.

  • Make in-context edits as you continue to build while everything else stays steady.
  • Small changes add up—each pass moves the app closer to the desired outcome.
  • Turn on Code view to see how your app is being built (without impacting the process).
  • Add existing tables from the Data tab to shape app behavior.
8
Review the splash widget

The splash widget showcases capabilities of the Vibe designer interface while agents build the plan, data model, and React app.

9
Review the generated Device Catalog App
Tip: Apps generated through generative AI will have variance—iteration matters to reach an acceptable outcome.

Add screenshots to assets/img/lab2/ and reference them here as you capture your “expected” app variant.

10
Open Plan view and review plan artifacts

Review the problem statement, personas, data model, and supporting agents.

Note: You can edit user roles and data model to align with business requirements.
Tip: User roles are conceptual only—Dataverse security roles control access (RBAC) after build-out.
11
Open Data Model view and review generated tables
Note: Generated tables/schemas/relationships can vary (AI-generated). Update via data agent or manually.
12
View Device table sample data
  1. Select the Device table
  2. Ellipsis (…) → View Data
  3. Confirm ~5 rows of sample data and review columns
13
Add ~15 additional rows of sample data
PROMPT 
Add 15 rows of sample data to the Device table
Tip: Sample data is critical for validating UX and business logic during development.
14
Remove Employee table

Next you’ll bring in data from an existing Dataverse table. Remove the Employee table from the model.

15
Add the existing User table
  1. Select Add Table
  2. Select Existing Dataverse table
16
Search and add User table
  1. Search for User
  2. Select it
  3. Click Add Selected
17
Create relationship: Requesting User lookup
PROMPT 
Add a new column called “Requesting User” to the Device Request Table. This column should look up to the User table so that a user can be selected as the person requesting the device.
Tip: When you create a 1-to-many relationship, a lookup column is added to the many-side table.
18
Synchronize app after data model changes

After updating the data model, click the App icon to synchronize changes.

19
Replace Employee table references with User table
PROMPT 
Update the Device Ordering App artifact to replace Employee Table with User Table
20
Add light/dark mode toggle (sun/moon icons)
PROMPT 
Add a light/dark mode toggle to the app. Add a toggle button (e.g. a sun/moon icon) in the top navigation bar that lets the user switch between light and dark mode.
Tip: If you want to revert changes, prompt the agent to undo the last action.
21
Save the application

Select Save.

22
Publish the application

Select Publish.

Note: Draft tables will be published to Dataverse during publish.
Tip: The first publish takes longest because tables are created in Dataverse schema. Subsequent publishes are faster.
23
Add a dashboard and set it as default landing page
PROMPT 
Add a dashboard page as the default landing page before the device catalog. Include summary cards for total devices, total requests, pending requests, and most requested device. Add a bar chart for requests by category and a line chart for requests over time.
Iteration note: If charts aren’t vibrant enough, continue refining via prompts (see next step).
24
Make bar chart more vibrant (and dark-mode friendly)
PROMPT 
Make the Requests by Category chart more colorful. Make sure it is legible in dark mode as well.
25
Use Picker tool for quick inline edits
  1. Select the Picker tool icon to enable inline edits
  2. Hover elements — select to see edit options
  3. Select Dashboard title — ask agent changes, adjust typography/colors, or directly type text
26
Add “New Request” button with dialog form
PROMPT 
Add a "New Request" button on the My Requests page, placed in the top right area. When clicked, it should open a dialog with a form that lets the user request a device.
27
Approve/Reject requests and update inventory on approval
PROMPT 
Let user approve or reject a pending device request. When clicking on a pending request tile, open a modal dialog which will allow user to review the request. Provide an Approve or Reject button on the bottom of the dialog. If a device request is approved, it should update the available quantity of that device in the device catalog.
Validate that approving the request results in an update to the available quantity of the device as well.
28
Add Low Stock Alert list/grid to dashboard
PROMPT 
Add a Low Stock list view on the Dashboard screen, where any device with fewer than 3 items on hand are listed. It should sort items by available quantity, with lowest at the top.
29
Publish again

Publish your solution again after these updates.

30
Share the app with end users + assign security roles
  1. Select the Share icon
  2. Select users or security groups
  3. Assign appropriate security roles
Tip: You can create custom security roles directly within your solution. See: Role-based security roles for Dataverse
31
Verify everything is solution-aware (ALM ready)

Go back to make.powerapps.com and open your custom solution. Confirm Vibe components (plan, plan artifacts, etc.) exist in solution context and are ready for ALM (pipelines, source control, downstream environments).

✅ Congratulations! You’ve successfully completed the Vibe Power Apps Lab.
